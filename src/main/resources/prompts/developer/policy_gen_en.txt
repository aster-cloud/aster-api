## Syntax Constraints
1. Every statement ends with a period (.)
2. Start with Module declaration: Module <name>.
3. Define record types: Define <TypeName> has <field1>, <field2>.
4. Define enums: Define <TypeName> as one of <A>, <B>, <C>.
5. Rules: Rule <name> given <params>, produce:
6. Local variables: Let <name> be <expr>.
7. Conditionals: block If/Otherwise with indentation (NOT inline if/then/else)
8. Pattern matching: Match/When (NOT match/case)
9. Construct instances: <TypeName> with <field> set to <value>.
10. Comparisons: equals to, greater than, less than, >=, <= (NOT "at least" or "at most")
11. Arithmetic: plus, minus, times, divided by (or symbolic: +, -, *, /)

## Common Mistakes to Avoid
- WRONG: Data UserInfo: has name : Text.
  RIGHT: Define UserInfo has name.
- WRONG: if age > 18 then "adult" else "minor"
  RIGHT: If age greater than 18\n  Return "adult".\nOtherwise\n  Return "minor".
- WRONG: match x: case A => result
  RIGHT: Match x:\n  When A, Return result.
- WRONG: Return result  (missing period)
  RIGHT: Return result.
- WRONG: Define User has name: Text, age: Int.
  RIGHT: Define User has name, age.
